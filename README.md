# python-beginner-workshop
Week1Task is a game where a random number will be generated and u need to guess it.
This game was made using python in the programming roadmap workshop


Python Beginner Workshop: Week 1 Documentation
1. Overview & Concepts Learned
This week focused on Python fundamentals. I set up my environment (VS Code & Python)
and learned how to use:
Variables & Data Types: Storing data and using input() and print().
Arithmetic & Conditions: Using math operators and if/elif/else for decision-making Loops: Using for and while loops to repeat actions.
Data Structures: Storing multiple items using Lists and Key-Value pairs in Dictionaries.
2. Main Task: Guess the Number Game
The final project was building a number guessing game. The program generates a random number from 1 to 100, and the user keeps guessing until they get it right.How it works: I used a while loop to keep the game running, if/else conditions to tell the user if their guess was "Higher" or "Lower," and variables to track the number of attempts.
3. Personal Understanding
What went well: I found learning the basic syntax for variables and if/else statements very straightforward.
What was challenging: Combining the while loops with the conditions in the final project was initially tricky to keep from creating an infinite loop.
Key Takeaway: I now feel confident writing and running basic Python scripts that take user input and make logical decisions.


Python Beginner Workshop: Week 2 Documentation
1. Overview & Concepts Learned
This week transitioned from basic scripting to writing organized, reusable, and professional code. I set up version control and learned about Object-Oriented Programming (OOP):
Version Control (GitHub): Creating repositories, writing README files, and making commits to save and manage code.
Functions & Refactoring: Defining functions with def, using parameters and return values, and organizing code using a main() function to avoid repetition.
Object-Oriented Programming (OOP): Creating class structures, initializing attributes with __init__, understanding the self keyword, and creating objects.
Class Methods & Interactive Systems: Adding behavior to classes via methods and combining them with while loops to create interactive command-line menus.
2. Main Task: Mini OOP Project
The final project for Week 2 was building a modular, CLI-based system using OOP principles (I built the Simple Budget Tracker).
How it works: I created a class to store the user's balance as an attribute. I defined class methods to add_income(), add_expense(), and show_balance(). Finally, I used a while loop to create an interactive menu that allowed the user to continuously call these methods until they chose to exit.
3. Personal Understanding
What went well: Learning how to use functions was a game-changer. It made refactoring my old code much easier and made my programs look significantly cleaner.
What was challenging: Wrapping my head around OOP concepts—especially remembering to use the self keyword everywhere to access class attributes and methods—took some practice to get used to.
Key Takeaway: I now feel comfortable structuring larger programs using classes and objects, and I understand how to safely back up my projects using GitHub.
